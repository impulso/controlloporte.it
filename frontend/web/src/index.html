Verifica se il DNS dinamico è aggiornato

                            

                                Il controllo DDNS confronta l'IP pubblico del visitatore con l'indirizzo IPv4 a cui
                                risolve il nome host inserito. Se i due indirizzi coincidono, il nome host DNS dinamico
                                sta puntando alla connessione da cui stai navigando.
                            

                            

                                È utile quando un accesso remoto, una telecamera, un NAS o un servizio domestico non
                                risponde e vuoi capire se il problema nasce dal DNS dinamico non aggiornato oppure da
                                port forwarding, firewall, NAT o CGNAT.
                            

                        


                        
Come leggere il risultato

                        

                            Se il controllo indica DDNS corretto, il nome host risolve allo stesso IP pubblico rilevato
                            dal sito. Se indica DDNS non allineato, il nome host punta a un IP diverso: il client DDNS
                            potrebbe non avere aggiornato il record, oppure stai navigando da una rete diversa.

Domande frequenti

                    

                        <% if (htmlWebpackPlugin.options.PAGE_MODE === 'nat') { %>
                        

                            Posso scegliere un IP da scansionare?
                    

                        No. La scansione riguarda solo l'IP pubblico rilevato dalla visita corrente.
                    

                        

                        

                            Il test controlla tutte le porte?
                    

                        No. Il profilo rapido controlla le 50 porte TCP più comuni.
                    

                        

                        

                            Se non trova porte aperte cosa significa?
                    

                        Significa solo che nessuna delle 50 porte TCP più comuni risulta aperta dall'esterno in questo momento.
                    

                        

                        <% } else if (htmlWebpackPlugin.options.PAGE_MODE === 'ddns') { %>
                        

                            Cosa controlla il test DDNS?
                    

                        Risolve il nome host inserito e confronta l'indirizzo ottenuto con l'IP pubblico rilevato per
                        la visita corrente. Serve a capire se il record DNS dinamico punta alla connessione giusta.
                    

                        

                        

                            Perché il DDNS risulta non allineato?
                    

                        Il client DDNS potrebbe non aver aggiornato il record, il router potrebbe usare un IP diverso,
                        oppure la connessione potrebbe essere dietro NAT o CGNAT. In alcuni casi stai semplicemente
                        facendo il test da una rete diversa da quella associata al nome host.
                    

                        

                        

                            Se il DDNS è corretto la porta deve risultare aperta?
                    

                        No. Il DDNS corretto conferma solo che il nome host punta all'IP pubblico rilevato. Per la
                        raggiungibilità del servizio devi controllare anche porta, firewall, port forwarding e servizio
                        in ascolto.
                    

                        

                        <% } else { %>
                        

                            Perché una porta risulta chiusa?
                    

                        Una porta può risultare chiusa se il servizio non è in ascolto, il firewall blocca la connessione,
                        il port forwarding non è configurato correttamente o l'indirizzo IP pubblico non è quello atteso.
                        Per una diagnosi passo passo, segui la
                        checklist per capire perché una porta risulta chiusa.
                    

                        

                        

                            Il test funziona anche per il port forwarding?
                    

                        Sì. Il controllo viene eseguito dall'esterno, quindi è utile per verificare se una regola di
                        port forwarding sul modem/router rende effettivamente raggiungibile il servizio.
                    

                        

                        

                            Posso controllare più porte insieme?
                    

                        Sì. Inserisci più porte separate da virgola, ad esempio 80, 443, 22.
                    

                        

                        

                            Posso usarlo per controlli di sicurezza legati alla NIS2?
                    

                        Sì, come controllo rapido dell'esposizione dei tuoi sistemi pubblici. Inserisci l'indirizzo del
                        server da verificare e usa il gruppo Accesso remoto per controllare se porte come SSH, Desktop
                        remoto o VNC risultano aperte verso Internet. Non sostituisce un audit NIS2 completo, ma aiuta a
                        individuare servizi amministrativi esposti al mondo. Il servizio applicativo non conserva log
                        delle verifiche effettuate, quindi puoi eseguire controlli puntuali senza creare uno storico
                        consultabile degli host e delle porte testate.
                    

                        

                        

                            Conservate i dati delle verifiche?
                    

                        No. Il servizio applicativo non conserva uno storico delle verifiche effettuate, degli host o
                        delle porte testate. Usiamo i dati inseriti solo per eseguire il controllo richiesto.
