Domande frequenti

Perché una porta risulta chiusa?

Una porta può risultare chiusa se il servizio non è in ascolto, il firewall blocca la connessione, il port forwarding non è configurato correttamente o l'indirizzo IP pubblico non è quello atteso. Per una diagnosi passo passo, segui la checklist per capire perché una porta risulta chiusa.

Il test funziona anche per il port forwarding?

Sì. Il controllo viene eseguito dall'esterno, quindi è utile per verificare se una regola di port forwarding sul modem/router rende effettivamente raggiungibile il servizio.

Posso controllare più porte insieme?

Sì. Inserisci più porte separate da virgola, ad esempio 80, 443, 22.

Posso usarlo per controlli di sicurezza legati alla NIS2?

Sì, come controllo rapido dell'esposizione dei tuoi sistemi pubblici. Inserisci l'indirizzo del server da verificare e usa il gruppo Accesso remoto per controllare se porte come SSH, Desktop remoto o VNC risultano aperte verso Internet. Non sostituisce un audit NIS2 completo, ma aiuta a individuare servizi amministrativi esposti al mondo. Il servizio applicativo non conserva log delle verifiche effettuate, quindi puoi eseguire controlli puntuali senza creare uno storico consultabile degli host e delle porte testate.

Conservate i dati delle verifiche?

No. Il servizio applicativo non conserva uno storico delle verifiche effettuate, degli host o delle porte testate. Usiamo i dati inseriti solo per eseguire il controllo richiesto.
